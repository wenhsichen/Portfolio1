OVERVIEW

Developed a graphical user interface for Massmine to improve user experience and expand target audiences

Massmine is a social media data collection tool. The initial version of Massmine website was a command-line based tool that was challenging for users to get started. Our project goal is to create a web application with graphic user interface for Massmine, making it more approachable and expanding its target audiences.

RESEARCH

User expectations for a comprehensive data collection and data visualization software

Stakeholder Interview

From the interview with stakeholders, we learned that the target audience for the application are tech-novice researchers who would require a user-friendly and streamlined graphical user interface. The following are features and information we would provide on the user interface.

  • Provide an easy-to-use input for queries
  • Provide researchers an overview of collected data
  • Provide data exploration through visuals
  • Provide researchers with the ability to filter through data
  • Provide researchers with the ability to export data

ITERATION

Got positive feedback from the usability testing and identified future improvement.

After we created a clickable prototype of the graphic user interface of Massmine, we conducted a usability test to examine whether users could interact with our design successfully, which included both qualitative and quantitive methods. Participants were asked to accomplish five tasks related to the primary functions, including the dashboard, navigation, adding projects, exporting data, and data visualization. Through the task, participants were asked to “Think Aloud” so that we could understand their thoughts. After that, participants were asked to complete two post-task questionnaires: the User Engagement Scale test (UES) and the PAAS subject rating scale, which were used to measure participants’ mental load and engagement with the prototype.


The participants felt that our overall design was clear and intuitive, but some labels needed to be more specific. Also, participants were not clear about the filter in the data visualization part. Participants also provided some recommendations, such as making it cloud-based so they could cooperate with their coworkers. Based on the issues found in the usability test, we redesigned the prototype.
